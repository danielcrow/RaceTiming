Content Security Policy (CSP) Errors - Browser Extensions
==========================================================

The CSP errors you're seeing in the browser console are NOT from the RaceTiming application.

They are from browser extensions that inject JavaScript into web pages:
- Zoom extension
- Amplitude analytics
- LaunchDarkly feature flags
- Other browser extensions

These errors are normal and do not affect the functionality of the RaceTiming application.

Evidence:
---------
1. The errors reference files like:
   - main.bundle.js
   - polyfills.bundle.js  
   - 5236.c4eaee8deaa930fe.js
   - 3350.f2c262284f303491.js
   
   These are NOT files in the RaceTiming application.

2. The errors mention:
   - "TokenRefreshInterceptor"
   - "LaunchDarkly client initialized"
   - "Amplitude Logger"
   - Zoom user IDs and company IDs
   
   These are all from browser extensions.

3. The master control page is working correctly:
   - HTTP 200 response confirmed
   - Route is properly registered
   - Template renders successfully

How to Verify:
--------------
1. Try the page in an incognito/private window (extensions disabled)
2. Disable browser extensions temporarily
3. Check the Network tab - you'll see the page loads successfully
4. The master control interface should be fully functional
